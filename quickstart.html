Quickstart Guide

This quick-start guide is designed to get you up and running as quickly as possible.

Step 1: Install SALVE

The recommended way to use SALVE is to fork it and layer your own configuration on top of it. By doing this, it becomes possible to trivially pull in upstream changes from the stable branch or release versions of SALVE. It also guarantees that whenever you put your configuration onto a machine, you will be able to deploy it with SALVE.

However, if you want to get started as quickly as possible, you should just install the latest stable version through easy_install or pip. You can look into forking the repo or handling inclusion of the SALVE source with your configuration by other means in the future. By way of example, you can install with pip:

pip install salve

Note that you must install salve on the machine that will be the target of your deployment, not necessarily the machine on which you are writing your manifests.

Step 2: Add Your Bashrc

These steps work with any file of your choice, but a bashrc is a good example of a file that should be managed with SALVE. You can use SALVE with any VCS, but we will assume that you have chosen git because of the numerous high-profile free-hosting options.

First, add your bashrc file to your repository. If you have forked SALVE, it is good practice to keep your files separate from the SALVE source and tests, to avoid merge conflicts with the upstream versions. To do this, we recommend using a top-level directory named "files". In your shell, inside of the repository root,

mkdir -p files/bash
cp ~/.bashrc files/bash/bashrc

Now, add a manifest, metaconfiguration that describes where the bashrc file goes on your system.

echo 'file {
source "files/bash/bashrc"
target "$HOME/.bashrc"
action "copy"
}' > bash.manifest

Finally, add these files to your git repository. git commit -am "Adding bashrc and bash manifest." Push to any remotes you use, and you are now ready to deploy your bashrc with SALVE.

Step 3: Deploy

The final step is to deploy your bashrc using SALVE. Since SALVE automagically backs up files that it overwrites, you can always recover the older bashrc file after you deploy. Just run

git clone https://my-remote/my-config
salve deploy --manifest my-config/bash.manifest

SALVE will take care of the rest. If you ever need to recover from the backups, they should be located in $HOME/.salve/backups/files and logged in $HOME/.salve/backup.log. Backed up files are named by a sha512 hash of their contents, but the backup log records the original locations of the backup file copies.
